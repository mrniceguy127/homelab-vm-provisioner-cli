Config Module¶

Helpers for reading user configs and persisting VM state.

Return the effective provisioner data root.

Relative PROVISIONER_DATA_DIR values resolve from the CLI project root.

homelab_vm_provisioner.config.load_config_from_stdin()[source]¶

Load a YAML configuration from stdin.

This function enables config piping and memory-based configurations without requiring temporary files.

Returns:

Parsed YAML document, or None if stdin is empty.

Return type:

dict

Raises:

yaml.YAMLError – If the input is not valid YAML.

homelab_vm_provisioner.config.resolve_config_path(config_path)[source]¶

Resolve a user-supplied config path to an existing file.

Supports explicit file names, extensionless names, and the config/ shorthand that maps to configs/.

Parameters:

config_path – User-supplied config path or shorthand.

Returns:

Existing config file path.

Return type:

Path

Raises:

FileNotFoundError – If no candidate path exists.
